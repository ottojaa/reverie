## Entries

- `2026-07-13` тАФ LLM provider is **Anthropic (Claude)**, not OpenAI. Single integration seam: `apps/backend/src/llm/anthropic.client.ts` (all LLM calls go through it). Models are env-configured: organize тЖТ `ANTHROPIC_ORGANIZE_MODEL` (Claude Sonnet 5, agentic tool-calling); per-document summary + vision тЖТ `ANTHROPIC_SUMMARY_MODEL`/`ANTHROPIC_VISION_MODEL` (Claude Haiku 4.5, high-volume/cheap). `ANTHROPIC_EFFORT` sets organize thinking depth.
- `2026-07-13` тАФ Organize (`services/organize.service.ts`) uses the Anthropic Messages API, which is **stateless** (no OpenAI `previous_response_id`). Multi-turn state (message history + `group_id` тЖТ document-id stash) lives in Redis via `services/organize-conversation.store.ts`, keyed by the opaque `response_id` the web client round-trips (TTL 1h). The system prompt + tool defs are cached via `cache_control`.
- `2026-07-13` тАФ Organize perf: `find_documents` never returns raw document UUIDs to the model тАФ it hands back short `group_id`s and stashes the ids server-side; `propose_organization` references groups by `group_id`. Keeps context/token cost small on the stateless API. Don't reintroduce UUID lists into tool output.
- `2026-07-13` тАФ Package manager is **Yarn 4 (Berry)** (migrated off pnpm, then off Yarn Classic). Pinned in-repo at `.yarn/releases/yarn-4.17.1.cjs` + `.yarnrc.yml` (`nodeLinker: node-modules`), so **corepack is never needed** тАФ in Docker/CI invoke it as `node .yarn/releases/yarn-4.17.1.cjs <cmd>`. Internal deps use `workspace:*`. `resolutions` pins the `nx`/`@nx/*` ecosystem to 22.4.0 (mismatched plugin minors break the project graph with a `Target.inputs` error). `.yarnrc.yml` sets `npmMinimalAgeGate: 14d` тАФ a supply-chain cooldown that refuses package versions younger than 14 days (Yarn auto-picks the newest older release per range; only exact transitive pins that are too new need a resolution or a loosened range, e.g. `@anthropic-ai/sdk` is `^0.107.0` not `^0.111.0`). If a fresh `yarn install` errors with "All versions satisfying X are quarantined", that's the age gate, not npm.
- `2026-07-13` тАФ Docker prod install uses Berry's `yarn workspaces focus @reverie/backend --production` in the builder (Debian `node:22`, for native-module ABI parity with the `node:22-slim` runtime), then copies the focused `node_modules` + `apps/backend/dist` + `libs/shared` into the runtime (no install there). `@reverie/shared` is a symlink in `node_modules` тЖТ `libs/shared`, so `libs/shared` (built) must be present at that relative path. Local Docker validation on Apple Silicon fails only at the PaddleOCR pip step (paddlepaddle wheels are x86_64-only); the prod host is x86_64 so that's fine.
